Privacy Policy

Last updated: June 2025

1. Overview

Datvex Prompt LAB ("we", "our", or "us") is a fully static, open-source web application. We are committed to protecting your privacy. This policy explains what information we collect, how we use it, and your rights.

2. Information We Collect

Datvex Prompt LAB does not collect, store, or transmit any personal information. The application runs entirely in your browser as a static site. The only data stored is your selected interface language, saved locally in your browser's localStorage.

3. Data Fetching

Prompt data, categories, and tags are fetched from GitHub's public raw content CDN at runtime. This is a read-only operation. No data you enter or interact with is sent to any server controlled by Datvex.

4. Third-Party Services

When you choose to send a prompt to an AI model (ChatGPT, Claude, Gemini, etc.), you are redirected to that third-party service. Their respective privacy policies apply. We do not share your data with these services — you initiate the interaction directly.

5. Cookies & Tracking

We do not use cookies, tracking pixels, analytics scripts, or any third-party tracking technologies. Your browsing activity within Datvex Prompt LAB is entirely private.

6. Session Storage

To improve performance, fetched prompt and category data is cached in your browser's sessionStorage. This data is automatically cleared when you close your browser tab and never leaves your device.

7. Changes to This Policy

We may update this Privacy Policy from time to time. Any changes will be reflected on this page with an updated date. We encourage you to review this policy periodically.

8. Contact

If you have any questions about this Privacy Policy, please open an issue on our GitHub repository.

Datvex Prompt LAB

A professional, structured repository of high-quality prompts designed for specialized tasks across multiple AI systems.

Overview

Welcome to the Datvex Prompt LAB — a professional, structured repository containing high-quality English-language prompts designed for specialized tasks across multiple AI systems.

The collection includes carefully curated prompts gathered from various sources across the global AI community — including GitHub repositories, Discord channels, and other professional resources — as well as original prompts developed specifically for this LAB.

The project ships as a static single-page application (SPA) built with vanilla HTML, CSS, and JavaScript. It fetches prompt data from JSON files hosted on GitHub and renders an interactive, filterable, searchable catalog with direct AI model integration.

Features

Feature Description
Structured Library Prompts organized by type and detailed categories
Multi-Model Launch Send prompts to ChatGPT, Claude, Grok, Gemini, DeepSeek, and more
11-Language UI Full interface localization (EN, RU, ZH, ES, etc.)
Real-time Search Instant prompt search with debounced input
Tag System Color-coded tags with count badges and sidebar filtering
Session Caching Data cached in sessionStorage

Project Structure

Datvex-prompt-LAB/

├── style.css               # Custom styles, animations, components
├── script.js               # Application logic, state, AI integration
│
├── data/
│   ├── prompts.json        # All prompt entries (fetched at runtime)
│   ├── categories.json     # Category groups and items
│   └── tags.json           # Tag definitions

Getting Started

Prerequisites

  • A modern web browser (Chrome 90+, Firefox 88+, Safari 14+)
  • A local HTTP server (for development)
  • No build tools, bundlers, or package managers required

Running Locally

# Clone the repository
git clone https://github.com/Datvex/Datvex-prompt-LAB.git

# Using Python 3
python -m http.server 8000

# Using Node.js
npx serve .

Architecture

The application follows a vanilla SPA architecture with no framework dependencies. All UI interactions, routing, and state management are handled with plain JavaScript.

Data is fetched from GitHub raw content URLs at runtime and cached locally to ensure lightning-fast subsequent loads and minimal network overhead.

Data Schemas

{
    "title": "Senior React Developer",
    "description": "Act as a senior React developer...",
    "prompt": "You are a senior React developer with 10+ years...",
    "category": "Programming & Tech",
    "category_id": "programming",
    "tags": ["react", "frontend", "coding"]
}

CSS Architecture

The UI follows a minimalist dark theme utilizing Tailwind CSS utility classes alongside custom CSS for specific animations, layout containment, and custom scrollbars.

Performance

  • Lazy loading: Prompts are loaded in batches via IntersectionObserver.
  • Event delegation: Single click listeners on the document root for dynamic elements.
  • CSS containment: contain: layout style is used on heavily rendered elements.
